# Curbie Smart Bin

This directory contains a self-contained implementation of the smart-bin workflow, including its own interface definitions.

## What Lives Where

- DICE workspace: `/afs/inf.ed.ac.uk/user/s25/s2552017/Curbie_Smart_Bin/dice_ws`
- TurtleBot workspace template: `/afs/inf.ed.ac.uk/user/s25/s2552017/Curbie_Smart_Bin/robot_ws_template`

- Optional Pico firmware copy: `/afs/inf.ed.ac.uk/user/s25/s2552017/Curbie_Smart_Bin/firmware/pico_main.py`

## Packages

### DICE side

- `bin_collection_msgs`
  - local action / message / service interfaces used by the operator and vision nodes

- `curbie_smart_bin_operator`
  - green/black Tk UI
  - manual driving
  - multi-path recording and storage
  - replay of any saved path
  - interrupt replay or seek with `X`
  - one-second sonar display
  - rename and delete saved paths
  - records seek events instead of recording the seek motion itself

### TurtleBot side

- `curbie_smart_bin_robot`
  - brings up the base
  - starts the Pico servo driver
  - starts the front and rear Logitech cameras
  - publishes the 5 Grove sonar distances on `/curbie/sonar_distances`

- `curbie_smart_bin_vision`
  - runs on the TurtleBot (not DICE) to reduce network sensitivity
  - rear-camera pickup alignment for markers `10` and `11`
  - front-camera dropoff alignment for marker `40`
  - pickup stand-off set to about `0.15 m`
  - dropoff stand-off set to about `0.32 m`
  - each successful dropoff increases the left offset by `0.50 m`

## Hardware Assumptions

- TurtleBot hostname: `SNOVER`
- DICE hostname: `hecarim`
- pickup markers: `10`, `11`
- collection marker: `40`
- front camera topic: `/curbie/front_camera/image_raw`
- rear camera topic: `/curbie/rear_camera/image_raw`
- camera roles are swapped intentionally:
  - the physical rear Logitech camera now publishes on `/curbie/front_camera/image_raw`
  - the physical front Logitech camera now publishes on `/curbie/rear_camera/image_raw`
- sonar pin mapping:
  - sonar 1: pin `5`, right / East
  - sonar 2: pin `16`, front-left / North-West
  - sonar 3: pin `18`, left / West
  - sonar 4: pin `22`, front-right / North-East
  - sonar 5: pin `24`, front / North

## Path Recording Model

Recorded paths are stored as JSON in `~/.ros/curbie_smart_bin/paths`.

Each path is a sequence of steps:

- motion step
  - relative odometry waypoints while you drove manually
- event step
  - `pickup_seek` for marker `10` or `11`
  - `dropoff_seek` for marker `40`

When you trigger a seek while recording, the operator:

1. closes the current motion segment
2. stores the seek command as an event
3. runs the seek
4. resumes recording from the new post-seek pose

That means the path file stores the command to seek, not the detailed robot motion produced by the seek controller.

## Replay Model

When replaying:

1. motion segments are followed with odometry-relative waypoint tracking
2. the latest sonar distances are shown in the UI for telemetry only (no sonar-based obstacle override is applied)
3. `pickup_seek` events run the rear-camera aligner and then engage the hooks
4. `dropoff_seek` events run the front-camera aligner against marker `40`, apply the next `0.50 m` left offset, and then release the hooks

The first successful dropoff uses `0.00 m` lateral offset, the second uses `0.50 m`, the third uses `1.00 m`, and so on.

## Keyboard Controls

- `W`: forward
- `S`: backward
- `A`: turn left
- `D`: turn right
- `X`: stop movement and cancel replay / seek
- `V`: engage hooks
- `C`: disengage hooks
- `1`: pickup seek on marker `10`
- `2`: pickup seek on marker `11`
- `G`: dropoff seek on marker `40`
- `O`: quick-record with an autogenerated path name
- `P`: stop recording
- `Space`: replay the currently selected saved path
- `Q`: close the UI

## Build And Run

### 1. Robot side on SNOVER

The new robot workspace is already installed at:

`/home/ubuntu/Curbie_Smart_Bin`

To run it:

```bash
ssh ubuntu@SNOVER
cd /home/ubuntu/Curbie_Smart_Bin
```

That script:

## What The UI Shows

- `Drive` tab
  - movement buttons
  - hook controls
  - activity log
- `Paths` tab
  - create a named recording
  - stop recording
  - insert pickup/dropoff events while recording
  - list saved paths
  - rename paths
  - replay selected paths
  - delete paths
- `Seek + Sonar` tab
  - pickup buttons for markers `10` and `11`
  - dropoff button for marker `40`
  - reset collection offset
  - 5-sonar readout refreshed every second

## Suggested Physical Test Flow

1. Start the robot script on `SNOVER`.
2. Start the DICE script.
3. Put the robot in a repeatable home pose.
4. Drive manually to a good staging pose for bin marker `10`.
5. In the `Paths` tab, enter a path name like `bin10_collection_demo`.
6. Click `Start Recording`.
7. Drive the route segments manually.
8. When you want pickup alignment to happen, click `Record + Run Pickup 10`.
9. Continue driving after the seek finishes.
10. When you want dropoff alignment to happen, click `Record + Run Dropoff 40`.
11. Stop the recording.
12. Select that saved path in the library and replay it.
13. Press `X` at any time to interrupt replay or a seek.

To make a second path for marker `11`, repeat the same flow with `Record + Run Pickup 11`.
